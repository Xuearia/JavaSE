### 一、内部类

#### 1. 成员内部类

成员内部类是定义在另一个类的内部，作为外部类的一个成员存在。

**特点**：

- 可以访问外部类的所有成员，包括私有成员。
- 外部类要访问内部类的成员，需要创建内部类的对象。

**示例代码**：

```java
// 外部类
class Outer {
    private int outerVar = 10;

    // 成员内部类
    class Inner {
        public void show() {
            System.out.println("访问外部类的私有变量: " + outerVar);
        }
    }

    public void accessInner() {
        Inner inner = new Inner();
        inner.show();
    }
}

public class MemberInnerClassDemo {
    public static void main(String[] args) {
        Outer outer = new Outer();
        outer.accessInner();

        // 也可以直接创建内部类对象
        Outer.Inner inner = outer.new Inner();
        inner.show();
    }
}
```



#### 2. 静态内部类

静态内部类是使用 `static` 修饰的内部类。

**特点**：

- 只能访问外部类的静态成员。
- 可以通过外部类名直接访问静态内部类。

**示例代码**：

```java
// 外部类
class OuterStatic {
    private static int outerStaticVar = 20;

    // 静态内部类
    static class InnerStatic {
        public void show() {
            System.out.println("访问外部类的静态变量: " + outerStaticVar);
        }
    }
}

public class StaticInnerClassDemo {
    public static void main(String[] args) {
        // 直接创建静态内部类对象
        OuterStatic.InnerStatic innerStatic = new OuterStatic.InnerStatic();
        innerStatic.show();
    }
}
```

#### 3. 局部内部类

局部内部类是定义在方法或代码块内部的类。

**特点**：

- 只能在定义它的方法或代码块内部使用。
- 可以访问外部类的成员以及方法中的局部变量，但局部变量必须是 `final` 或事实上的 `final`（Java 8 及以后）。

**示例代码**：

```java
// 外部类
class OuterLocal {
    public void outerMethod() {
        int localVar = 30; // 事实上的 final

        // 局部内部类
        class InnerLocal {
            public void show() {
                System.out.println("访问外部类方法的局部变量: " + localVar);
            }
        }

        InnerLocal innerLocal = new InnerLocal();
        innerLocal.show();
    }
}

public class LocalInnerClassDemo {
    public static void main(String[] args) {
        OuterLocal outerLocal = new OuterLocal();
        outerLocal.outerMethod();
    }
}
```

#### 4. 匿名内部类

匿名内部类是一种没有名字的局部内部类，通常用于创建只使用一次的类。

**特点**：

- 没有类名，直接实现接口或继承抽象类。
- 只能使用一次。

**示例代码**：

```java
// 接口
interface MyInterface {
    void doSomething();
}

public class AnonymousInnerClassDemo {
    public static void main(String[] args) {
        // 匿名内部类实现接口
        MyInterface myInterface = new MyInterface() {
            @Override
            public void doSomething() {
                System.out.println("匿名内部类实现接口方法");
            }
        };

        myInterface.doSomething();
    }
}
```

### 二、枚举

#### 1. 认识枚举

枚举是一种特殊的类，用于定义一组固定的常量。

**示例代码**：

```java
// 枚举类
enum Weekday {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}

public class EnumDemo {
    public static void main(String[] args) {
        Weekday today = Weekday.MONDAY;
        System.out.println("今天是: " + today);
    }
}
```

#### 2. 枚举的应用场景

- **状态管理**：如订单状态（待支付、已支付、已发货等）。
- **选项选择**：如性别（男、女）。

### 三、泛型

#### 1. 认识泛型

泛型是 Java 提供的一种参数化类型的机制，允许在定义类、接口或方法时使用类型参数，提高代码的复用性和类型安全性。

**示例代码**：

```java
// 泛型类
class GenericClass<T> {
    private T data;

    public GenericClass(T data) {
        this.data = data;
    }

    public T getData() {
        return data;
    }
}

public class GenericDemo {
    public static void main(String[] args) {
        GenericClass<String> genericString = new GenericClass<>("Hello");
        System.out.println(genericString.getData());

        GenericClass<Integer> genericInt = new GenericClass<>(123);
        System.out.println(genericInt.getData());
    }
}
```

#### 2. 自定义泛型类

在类名后面使用 `<T>` 等类型参数来定义泛型类。

**示例代码**：

```java
// 自定义泛型类
class MyGenericClass<T> {
    private T value;

    public MyGenericClass(T value) {
        this.value = value;
    }

    public T getValue() {
        return value;
    }
}
```

#### 3. 自定义泛型接口

在接口名后面使用 `<T>` 等类型参数来定义泛型接口。

**示例代码**：

```java
// 自定义泛型接口
interface MyGenericInterface<T> {
    T getValue();
}

// 实现泛型接口
class MyGenericInterfaceImpl implements MyGenericInterface<String> {
    private String value;

    public MyGenericInterfaceImpl(String value) {
        this.value = value;
    }

    @Override
    public String getValue() {
        return value;
    }
}
```

#### 4. 泛型方法

在方法的返回类型前面使用 `<T>` 等类型参数来定义泛型方法。

**示例代码**：

```java
class GenericMethodClass {
    public <T> void printArray(T[] array) {
        for (T element : array) {
            System.out.print(element + " ");
        }
        System.out.println();
    }
}

public class GenericMethodDemo {
    public static void main(String[] args) {
        GenericMethodClass genericMethodClass = new GenericMethodClass();
        Integer[] intArray = {1, 2, 3, 4, 5};
        String[] stringArray = {"Hello", "World"};

        genericMethodClass.printArray(intArray);
        genericMethodClass.printArray(stringArray);
    }
}
```

#### 5. 泛型限定

使用 `extends` 关键字对泛型类型进行限定，如 `<? extends Number>` 表示泛型类型必须是 `Number` 或其子类。

**示例代码**：

```java
class GenericLimitClass {
    public static void printNumberList(java.util.List<? extends Number> list) {
        for (Number num : list) {
            System.out.print(num + " ");
        }
        System.out.println();
    }
}

public class GenericLimitDemo {
    public static void main(String[] args) {
        java.util.List<Integer> intList = java.util.Arrays.asList(1, 2, 3);
        java.util.List<Double> doubleList = java.util.Arrays.asList(1.1, 2.2, 3.3);

        GenericLimitClass.printNumberList(intList);
        GenericLimitClass.printNumberList(doubleList);
    }
}
```

#### 6. 泛型擦除

泛型擦除是 Java 泛型实现的一种机制，在编译时编译器会将泛型类型信息擦除，替换为原始类型（如果没有限定则是 `Object`，如果有限定则是限定类型）。下面为你展示编译前和反编译后的代码差异。

### 编译前的代码

```java
import java.util.ArrayList;
import java.util.List;

public class GenericErasureExample {
    public static void main(String[] args) {
        List<String> stringList = new ArrayList<>();
        stringList.add("Hello");
        String element = stringList.get(0);
        System.out.println(element);
    }
}
```

在上述代码中，`List<String>` 明确指定了列表存储的元素类型为 `String`。

### 反编译后的代码（模拟）

使用工具（如 `javap` 或反编译工具 JD-GUI）对编译后的 `.class` 文件进行反编译，会得到类似如下的代码：

```java
import java.util.ArrayList;
import java.util.List;

public class GenericErasureExample {
    public static void main(String[] args) {
        // 泛型被擦除，实际类型为 Object
        List stringList = new ArrayList();
        stringList.add("Hello");
        // 手动进行类型转换
        String element = (String) stringList.get(0);
        System.out.println(element);
    }
}
```

可以看到，编译后的代码中泛型类型信息被擦除，`List<String>` 变成了 `List`，也就是原始类型。在获取元素时，需要手动进行类型转换。
