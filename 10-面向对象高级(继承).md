##  一.继承入门 ##

 ### 1.继承的概念

继承是 Java 面向对象编程的核心特性之一，它通过 **类之间的层级关系**，实现代码复用与功能扩展。

- **本质**：子类（派生类）继承父类（基类）的属性和方法，形成 “**is-a**” 关系（如：狗是动物 → `Dog is a Animal`）。

- **作用**：
1. **代码复用**：避免重复编写父类已有的属性和方法。
  2. **功能扩展**：子类可新增属性或重写父类方法，实现差异化逻辑。
3. **层次结构**：形成清晰的类层级（如：动物 → 食草动物 / 食肉动物 → 兔子 / 狮子）。

### 2.**生活中的继承示例**

![](img/09/14B0951E-FC75-47A3-B611-4E1883887339.jpg)

**场景**：动物分类

- **父类**：`动物类`（通用属性：名称、年龄；通用行为：进食）。

- 子类：

  - `食草动物类`（继承动物类，新增行为：吃草）。
  - `食肉动物类`（继承动物类，新增行为：捕猎）。
  
- **关系**：食草动物和食肉动物 **is-a** 动物，但具有各自特有的行为。

### **3.Java 中继承的实现**

#### **语法格式**

```java
class 父类 {  
    // 父类属性和方法  
}  

class 子类 extends 父类 {  
    // 子类新增属性或重写方法  
}  
```

#### **关键代码示例**

```java
// 父类：动物类  
class Animal {  
    protected String name; // 受保护属性，子类可继承  
    public Animal(String name) { this.name = name; }  
    public void eat() { System.out.println(name + "正在进食"); }  
}  

// 子类：狗类（继承动物类）  
class Dog extends Animal {  
    private String breed; // 子类特有属性  
    public Dog(String name, String breed) {  
        super(name); // 调用父类构造器初始化继承的属性  
        this.breed = breed;  
    }  
    // 子类特有方法  
    public void bark() { System.out.println(name + "（品种：" + breed + "）正在汪汪叫"); }  
}  

// 测试类  
public class InheritanceDemo {  
    public static void main(String[] args) {  
        Dog dog = new Dog("旺财", "金毛");  
        dog.eat(); // 继承自父类的方法：旺财正在进食  
        dog.bark(); // 子类特有方法：旺财（品种：金毛）正在汪汪叫  
    }  
}  
```



### 二、继承的好处

#### 1. 提高代码复用性

当多个类存在相同的属性和方法时，可以将这些相同的部分抽取到父类中，子类继承父类后就可以直接使用这些属性和方法，避免了重复编写代码。

#### 2. 便于代码维护和扩展

如果需要修改或扩展父类的功能，只需要在父类中进行操作，所有子类都会受到影响，无需逐个修改子类代码。


| **优点**                             | **缺点**                                                |
| ------------------------------------ | ------------------------------------------------------- |
| 代码复用，减少冗余                   | 父类修改可能影响所有子类（需谨慎维护）                  |
| 类层次结构清晰，符合现实逻辑         | 过度使用继承可能导致类结构复杂（“is-a” 关系需严格遵循） |
| 支持多态特性（父类引用指向子类对象） | 单继承限制，无法同时继承多个类的特性                    |

#### 3. 示例代码

以下是一个简单的示例，展示了继承如何提高代码复用性。

```java
// 定义父类 People（抽取公共属性和行为）
class People {
    // 受保护的成员变量（子类可继承）
    protected String name; 
    protected int age; 

    // 父类构造方法
    public People(String name, int age) { 
        this.name = name;
        this.age = age;
    }

    // 公共行为：自我介绍
    public void introduce() { 
        System.out.println("我叫" + name + "，今年" + age + "岁。"); // 子类无需重复实现
    }
}

// 子类Student（继承People，扩展学生特有的属性和行为）
class Student extends People { 
    private String studentId; // 学生特有的学号属性

    // 子类构造方法（必须调用父类构造器初始化name和age）
    public Student(String name, int age, String studentId) { 
        super(name, age); // 调用父类有参构造器
        this.studentId = studentId; // 初始化子类属性
    }

    // 子类特有行为：学习
    public void study() { 
        System.out.println(name + "（学号：" + studentId + "）正在学习。"); // 复用父类的name，结合子类的studentId
    }
}

// 子类Teacher（继承People，扩展教师特有的属性和行为）
class Teacher extends People { 
    private String subject; // 教师特有的教授科目属性

    // 子类构造方法
    public Teacher(String name, int age, String subject) { 
        super(name, age); // 调用父类有参构造器
        this.subject = subject; // 初始化子类属性
    }

    // 子类特有行为：授课
    public void teach() { 
        System.out.println(name + "正在教授" + subject + "课程。"); // 复用父类的name，结合子类的subject
    }
}

public class InheritanceBenefits {
    public static void main(String[] args) {
        // 创建Student对象（自动拥有People的属性和方法）
        Student student = new Student("张三", 18, "001"); 
        student.introduce(); // 输出：我叫张三，今年18岁。（直接使用父类方法）
        student.study(); // 输出：张三（学号：001）正在学习。（子类扩展方法）

        // 创建Teacher对象
        Teacher teacher = new Teacher("李四", 30, "数学"); 
        teacher.introduce(); // 输出：我叫李四，今年30岁。（直接使用父类方法）
        teacher.teach(); // 输出：李四正在教授数学课程。（子类扩展方法）
    }
}
```

#### 4. 代码解释

- `People` 类是父类，包含了 `name` 和 `age` 属性以及 `introduce()` 方法。
- `Student` 类和 `Teacher` 类是子类，继承自 `People` 类，它们可以直接使用 `People` 类的属性和方法，同时还添加了自己的属性和方法。
- 在 `main` 方法中，创建了 `Student` 类和 `Teacher` 类的对象，并调用了它们的方法。

### 三、权限修饰符

#### 1. 权限修饰符种类及作用

| 修饰符                | 类内部 | 同一包内 | 子类（不同包） | 其他类（不同包） |
| --------------------- | ------ | -------- | -------------- | ---------------- |
| `private`             | ✅      | ❌        | ❌              | ❌                |
| `default`（无修饰符） | ✅      | ✅        | ❌              | ❌                |
| `protected`           | ✅      | ✅        | ✅（需继承）    | ❌                |
| `public`              | ✅      | ✅        | ✅              | ✅                |

### 说明：

- **排列逻辑**：从 **权限最严格**（`private`）到 **权限最开放**（`public`）依次排列。
- 核心差异：
  - `private`：完全封闭，仅限类内部访问。
  - `default`：包内可见，跨包不可见（包括子类）。
  - `protected`：包内可见 + 跨包子类可见（需继承）。
  - `public`：完全开放，无访问限制。

#### 2. 示例代码

```java
// 定义父类 Fu
class Fu {
    // 私有成员变量
    private int privateVar = 1;
    // 缺省成员变量
    int defaultVar = 2;
    // 受保护成员变量
    protected int protectedVar = 3;
    // 公开成员变量
    public int publicVar = 4;

    // 私有方法
    private void privateMethod() {
        System.out.println("私有方法");
    }

    // 缺省方法
    void defaultMethod() {
        System.out.println("缺省方法");
    }

    // 受保护方法
    protected void protectedMethod() {
        System.out.println("受保护方法");
    }

    // 公开方法
    public void publicMethod() {
        System.out.println("公开方法");
    }

    // 在本类中访问所有成员
    public void accessMembersInThisClass() {
        System.out.println("在本类中访问：");
        System.out.println(privateVar); // 输出: 1
        System.out.println(defaultVar); // 输出: 2
        System.out.println(protectedVar); // 输出: 3
        System.out.println(publicVar); // 输出: 4
        privateMethod(); // 输出: 私有方法
        defaultMethod(); // 输出: 缺省方法
        protectedMethod(); // 输出: 受保护方法
        publicMethod(); // 输出: 公开方法
    }
}

// 定义同一包中的类 SamePackageClass
class SamePackageClass {
    public void accessMembersInSamePackage() {
        Fu fu = new Fu();
        System.out.println("在同一包中访问：");
        // 不能访问私有成员
        // System.out.println(fu.privateVar);
        // fu.privateMethod();
        System.out.println(fu.defaultVar); // 输出: 2
        System.out.println(fu.protectedVar); // 输出: 3
        System.out.println(fu.publicVar); // 输出: 4
        fu.defaultMethod(); // 输出: 缺省方法
        fu.protectedMethod(); // 输出: 受保护方法
        fu.publicMethod(); // 输出: 公开方法
    }
}

// 定义不同包中的子类 DifferentPackageSubclass
class DifferentPackageSubclass extends Fu {
    public void accessMembersInDifferentPackageSubclass() {
        System.out.println("在不同包子类中访问：");
        // 不能访问私有成员
        // System.out.println(privateVar);
        // privateMethod();
        // 不能访问缺省成员
        // System.out.println(defaultVar);
        // defaultMethod();
        System.out.println(protectedVar); // 输出: 3
        System.out.println(publicVar); // 输出: 4
        protectedMethod(); // 输出: 受保护方法
        publicMethod(); // 输出: 公开方法
    }
}

// 定义不同包中的非子类 DifferentPackageNonSubclass
class DifferentPackageNonSubclass {
    public void accessMembersInDifferentPackageNonSubclass() {
        Fu fu = new Fu();
        System.out.println("在不同包非子类中访问：");
        // 不能访问私有成员
        // System.out.println(fu.privateVar);
        // fu.privateMethod();
        // 不能访问缺省成员
        // System.out.println(fu.defaultVar);
        // fu.defaultMethod();
        // 不能访问受保护成员
        // System.out.println(fu.protectedVar);
        // fu.protectedMethod();
        System.out.println(fu.publicVar); // 输出: 4
        fu.publicMethod(); // 输出: 公开方法
    }
}

public class PermissionModifiers {
    public static void main(String[] args) {
        Fu fu = new Fu();
        fu.accessMembersInThisClass();
        /*
        输出:
        在本类中访问：
        1
        2
        3
        4
        私有方法
        缺省方法
        受保护方法
        公开方法
         */

        SamePackageClass samePackageClass = new SamePackageClass();
        samePackageClass.accessMembersInSamePackage();
        /*
        输出:
        在同一包中访问：
        2
        3
        4
        缺省方法
        受保护方法
        公开方法
         */

        DifferentPackageSubclass differentPackageSubclass = new DifferentPackageSubclass();
        differentPackageSubclass.accessMembersInDifferentPackageSubclass();
        /*
        输出:
        在不同包子类中访问：
        3
        4
        受保护方法
        公开方法
         */

        DifferentPackageNonSubclass differentPackageNonSubclass = new DifferentPackageNonSubclass();
        differentPackageNonSubclass.accessMembersInDifferentPackageNonSubclass();
        /*
        输出:
        在不同包非子类中访问：
        4
        公开方法
         */
    }
}    
```

#### 3. 代码解释

- `Fu` 类中定义了四种不同权限修饰符的成员变量和方法。
- `SamePackageClass` 类在同一包中，只能访问 `Fu` 类中除私有成员外的其他成员。
- `DifferentPackageSubclass` 类是 `Fu` 类在不同包中的子类，只能访问 `Fu` 类中受保护和公开的成员。
- `DifferentPackageNonSubclass` 类是 `Fu` 类在不同包中的非子类，只能访问 `Fu` 类中公开的成员。

#### 4. 总结

合理使用权限修饰符可以控制类的成员的访问范围，提高代码的安全性和可维护性。

### 四、单继承、Object

#### 1. 单继承特性

![](img/09/1664016601977.png)

Java 语言只支持单继承，即一个子类只能有一个直接父类，不支持多继承（一个子类不能同时继承多个直接父类）。不过支持多层继承，例如 `C` 类可以继承 `B` 类，`B` 类继承 `A` 类。

**翻译过来就是**:

- **Java语言只支持单继承，不支持多继承，但是可以多层继承**。就像家族里儿子、爸爸和爷爷的关系一样：一个儿子只能有一个爸爸，不能有多个爸爸，但是爸爸也是有爸爸的。

![](img\09\java-extends-2020-12-08.png)

#### 2. 示例代码

```java
// 定义类 A
class A {
    public void methodA() {
        System.out.println("A 类的方法");
    }
}

// 定义类 B，继承自 A
class B extends A {
    public void methodB() {
        System.out.println("B 类的方法");
    }
}

// 定义类 C，继承自 B
class C extends B {
    public void methodC() {
        System.out.println("C 类的方法");
    }
}

public class SingleInheritance {
    public static void main(String[] args) {
        C c = new C();
        // 可以调用 A 类的方法
        c.methodA();
        // 可以调用 B 类的方法
        c.methodB();
        // 可以调用 C 类自己的方法
        c.methodC();
    }
}
```

#### 3. 代码解释

- `C` 类继承自 `B` 类，`B` 类继承自 `A` 类，形成了多层继承关系。
- `C` 类的对象可以调用 `A` 类、`B` 类和自己的方法。

#### 4. Object 类

`Object` 类是 Java 中所有类的祖宗类，如果一个类没有显式地继承其他类，那么它默认继承 `Object` 类。`Object` 类中定义了一些通用的方法，如 `toString()`、`equals()` 等，子类可以重写这些方法来满足自身需求。

#### 5. 示例代码

```java
// 定义类 Person，默认继承 Object 类
class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 重写 Object 类的 toString() 方法
    @Override
    public String toString() {
        return "Person [name=" + name + ", age=" + age + "]";
    }
}

public class ObjectClass {
    public static void main(String[] args) {
        Person person = new Person("张三", 20);
        // 调用重写后的 toString() 方法
        System.out.println(person);
    }
}
```

#### 6. 代码解释

- `Person` 类没有显式地继承其他类，默认继承 `Object` 类。
- `Person` 类重写了 `Object` 类的 `toString()` 方法，使其返回更有意义的信息。

#### 7. 总结

单继承保证了类的层次结构清晰，`Object` 类为所有类提供了一些通用的方法，子类可以根据需要重写这些方法。

### 五、方法重写

#### 1. 方法重写定义

当子类觉得父类的某个方法不能满足自身需求时，子类可以重新实现一个与父类方法名称、参数列表完全相同的方法，去覆盖父类的这个方法，这就是方法重写。

#### 2. 重写规则

- 重写的方法上面可以添加 `@Override` 注解，用于标注该方法是重写父类的方法，这样可以提高代码的可读性和安全性，让编译器帮忙检查方法重写是否正确。
- 子类重写父类方法时，访问权限必须大于或者等于父类方法的权限，权限大小关系为 `public > protected > 缺省`。
- 重写的方法返回值类型必须与被重写的方法返回值类型一样，或者范围更小（例如父类方法返回 `Object`，子类方法可以返回 `String`）。
- 私有方法、静态方法不能被重写，如果重写会报错。

#### 3. 应用场景及示例

方法重写常见的应用场景是子类根据自身的特点，对父类的方法进行定制化实现。以下是一个简单的示例：

```java
// 定义父类 Animal
class Animal {
    public void makeSound() {
        System.out.println("动物发出声音");
    }
}

// 定义子类 Dog，继承自 Animal
class Dog extends Animal {
    // 重写父类的 makeSound() 方法
    @Override
    public void makeSound() {
        System.out.println("汪汪汪");
    }
}

// 定义子类 Cat，继承自 Animal
class Cat extends Animal {
    // 重写父类的 makeSound() 方法
    @Override
    public void makeSound() {
        System.out.println("喵喵喵");
    }
}

public class MethodOverride {
    public static void main(String[] args) {
        Animal dog = new Dog();
        Animal cat = new Cat();

        // 调用重写后的方法
        dog.makeSound();
        cat.makeSound();
    }
}
```

#### 4. 代码解释

- `Dog` 类和 `Cat` 类继承自 `Animal` 类，并重写了 `Animal` 类的 `makeSound()` 方法。
- 在 `main` 方法中，创建了 `Dog` 类和 `Cat` 类的对象，并将它们赋值给 `Animal` 类型的引用变量，然后调用 `makeSound()` 方法，实际调用的是子类重写后的方法。

#### 5. 总结

方法重写允许子类根据自身需求修改父类方法的实现，提高了代码的灵活性和可扩展性。

### 六、子类中访问成员的特点

#### 1. 就近原则

在子类中访问成员变量和成员方法时，遵循就近原则，即优先访问子类自身定义的成员，如果子类中不存在该成员，再去父类中查找。

#### 2. 示例代码

```java
// 定义父类 Father
class Father {
    int num = 10;

    public void show() {
        System.out.println("父类的 show 方法");
    }
}

// 定义子类 Son，继承自 Father
class Son extends Father {
    int num = 20;

    // 重写父类的 show() 方法
    @Override
    public void show() {
        System.out.println("子类的 show 方法");
    }

    public void test() {
        System.out.println(num); // 输出 20，访问子类的 num
        show(); // 输出“子类的 show 方法”，调用子类的 show() 方法
        System.out.println(super.num); // 输出 10，访问父类的 num
        super.show(); // 输出“父类的 show 方法”，调用父类的 show() 方法
    }
}

public class SubclassMemberAccess {
    public static void main(String[] args) {
        Son son = new Son();
        son.test();
    }
}
```

#### 3. 代码解释

- `Son` 类继承自 `Father` 类，`Son` 类和 `Father` 类都定义了 `num` 变量和 `show()` 方法。
- 在 `Son` 类的 `test()` 方法中，直接访问 `num` 变量和调用 `show()` 方法时，遵循就近原则，访问的是子类的成员。
- 使用 `super` 关键字可以明确访问父类的成员。

#### 4. 使用 `this` 和 `super` 区分

- `this` 关键字用于引用当前对象，可以访问当前对象的成员变量和方法。
- `super` 关键字用于引用父类对象，可以访问父类的成员变量和方法。

#### 5. 总结

子类访问成员时遵循就近原则，使用 `this` 和 `super` 关键字可以明确指定访问的是子类还是父类的成员。

### 七、子类中访问构造器的特点

#### 1. 子类构造器默认调用父类构造器

子类的所有构造器，在执行时都会先调用父类的构造器（默认调用父类的无参构造器，即 `super()`），然后再执行自己的构造器代码。这是因为子类对象中包含了父类对象的部分，需要先初始化父类部分。

#### 2. 示例代码

```java
// 定义父类 Animal
class Animal {
    public Animal() {
        System.out.println("Animal 的无参构造器");
    }

    public Animal(String name) {
        System.out.println("Animal 的有参构造器，动物名字：" + name);
    }
}

// 定义子类 Dog，继承自 Animal
class Dog extends Animal {
    public Dog() {
        // 默认调用父类的无参构造器
        super();
        System.out.println("Dog 的无参构造器");
    }

    public Dog(String name) {
        // 手动调用父类的有参构造器
        super(name);
        System.out.println("Dog 的有参构造器");
    }
}

public class SubclassConstructorAccess {
    public static void main(String[] args) {
        System.out.println("创建无参 Dog 对象：");
        Dog dog1 = new Dog();

        System.out.println("创建有参 Dog 对象：");
        Dog dog2 = new Dog("旺财");
    }
}
```

#### 3. 代码解释

- `Dog` 类的无参构造器默认调用 `Animal` 类的无参构造器。
- `Dog` 类的有参构造器手动调用 `Animal` 类的有参构造器。

#### 4. 手动调用父类有参构造器

如果不想使用默认的 `super()` 方式调用父类构造器，而是想调用父类的有参构造器，可以手动使用 `super(参数)` 的形式，并且 `super(参数)` 必须位于子类构造器的第一行。

#### 5. 在本类中访问自己的构造方法

在一个类中，可以使用 `this()` 调用本类的无参构造器，使用 `this(参数)` 调用本类的有参构造器，同样，这种调用也必须位于构造器的第一行。

#### 6. 示例代码

```java
// 定义类 Person
class Person {
    private String name;
    private int age;

    public Person() {
        // 调用本类的有参构造器
        this("未知", 0);
        System.out.println("Person 的无参构造器");
    }

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
        System.out.println("Person 的有参构造器");
    }
}

public class ConstructorCallInClass {
    public static void main(String[] args) {
        Person person = new Person();
    }
}
```

#### 7. 代码解释

- `Person` 类的无参构造器使用 `this("未知", 0)` 调用本类的有参构造器。

#### 8. 总结

子类构造器会先调用父类构造器，确保父类部分的初始化。可以使用 `super` 关键字手动调用父类的有参构造器，使用 `this` 关键字在本类的构造器中调用其他构造器。

### 八、继承的总结

继承是 Java 面向对象编程中的一个核心特性，它为代码的复用和扩展提供了强大的支持。通过继承，子类能够轻松地获取父类的非私有属性和方法，避免了代码的重复编写，提高了开发效率。同时，方法重写允许子类根据自身需求对父类方法进行定制化，增强了代码的灵活性和可扩展性。

在使用继承时，需要注意以下几点：

1. **权限修饰符**：合理使用 `private`、缺省、`protected` 和 `public` 权限修饰符，控制类成员的访问范围，提高代码的安全性和可维护性。
2. **单继承**：Java 只支持单继承，一个子类只能有一个直接父类，但支持多层继承，这保证了类的层次结构清晰。
3. **方法重写规则**：重写方法时要遵循访问权限、返回值类型等规则，使用 `@Override` 注解提高代码的可读性和安全性。
4. **子类访问成员**：子类访问成员遵循就近原则，使用 `this` 和 `super` 关键字可以明确指定访问的是子类还是父类的成员。
5. **构造器调用**：子类构造器会默认调用父类的无参构造器，也可以手动调用父类的有参构造器，还可以在本类构造器中使用 `this` 关键字调用其他构造器。
